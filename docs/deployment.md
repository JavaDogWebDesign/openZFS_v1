# Deployment Guide

## Requirements

- **OS**: Debian 12 (Bookworm) or later
- **Kernel**: Must support ZFS (DKMS module installed via `zfsutils-linux`)
- **RAM**: 2 GB minimum (ZFS ARC + Python + Node build)
- **Disk**: ZFS pool disks + ~500 MB for the application

## Quick Install

```bash
# Clone or copy the project to the server
git clone <repo> /tmp/openzfs-manager
cd /tmp/openzfs-manager

# Run the installer as root
sudo bash deploy/install.sh
```

The installer handles everything: system packages, Python venv, frontend build, systemd service, nginx config, Samba/NFS setup.

## What the Installer Does

### System Packages Installed
| Package | Purpose |
|---------|---------|
| `zfsutils-linux` | ZFS kernel module + userspace tools (`zpool`, `zfs`) |
| `linux-headers-$(uname -r)` | Kernel headers for DKMS module compilation |
| `python3`, `python3-venv`, `python3-pip`, `python3-dev` | Python runtime and build tools |
| `build-essential`, `libffi-dev` | C compiler for native Python extensions (bcrypt) |
| `nodejs`, `npm` | Frontend build toolchain |
| `nginx` | Reverse proxy and static file server |
| `samba` | SMB/CIFS file sharing |
| `nfs-kernel-server` | NFS file sharing |
| `smartmontools` | Disk health monitoring (`smartctl`) |
| `rsync` | File synchronization during install |
| `openssl` | JWT secret key generation |

### Directory Layout (Post-Install)

```
/opt/openzfs-manager/
├── backend/              ← Python source code
├── frontend/
│   ├── src/              ← TypeScript source (only needed for rebuilds)
│   └── dist/             ← Built static files (served by nginx)
├── venv/                 ← Python virtual environment
└── data/
    └── openzfs.db        ← SQLite database (created on first start)

/etc/systemd/system/openzfs-manager.service  ← systemd unit
/etc/nginx/sites-available/openzfs-manager   ← nginx config
/etc/nginx/sites-enabled/openzfs-manager     ← symlink
/etc/samba/openzfs-shares.conf               ← Managed SMB shares (included by smb.conf)
/etc/exports.d/openzfs.exports               ← Managed NFS exports
```

## Configuration

### Environment Variables

All settings are controlled via environment variables with the `OPENZFS_` prefix. They are set in the systemd service file.

```bash
# Edit the service file
sudo systemctl edit openzfs-manager --full

# Or edit directly
sudo vim /etc/systemd/system/openzfs-manager.service
```

| Variable | Default | Description |
|----------|---------|-------------|
| `OPENZFS_SECRET_KEY` | (generated) | JWT signing key — auto-generated by installer |
| `OPENZFS_DATABASE_URL` | `sqlite+aiosqlite:////opt/openzfs-manager/data/openzfs.db` | Database path |
| `OPENZFS_ACCESS_TOKEN_EXPIRE_MINUTES` | `30` | JWT access token lifetime |
| `OPENZFS_REFRESH_TOKEN_EXPIRE_DAYS` | `7` | JWT refresh token lifetime |
| `OPENZFS_CORS_ORIGINS` | `[]` | CORS origins (empty = disabled, correct for nginx proxy) |
| `OPENZFS_WS_POOL_STATUS_INTERVAL` | `5` | Seconds between WebSocket broadcasts |
| `OPENZFS_COMMAND_TIMEOUT` | `30` | Subprocess timeout in seconds |
| `OPENZFS_DEBUG` | `false` | Enable SQLAlchemy echo logging |

### Regenerating the JWT Secret Key

```bash
# Generate a new key
NEW_KEY=$(openssl rand -hex 32)

# Update the service file
sudo sed -i "s/OPENZFS_SECRET_KEY=.*/OPENZFS_SECRET_KEY=${NEW_KEY}/" \
    /etc/systemd/system/openzfs-manager.service

sudo systemctl daemon-reload
sudo systemctl restart openzfs-manager
```

**Warning:** Changing the secret key invalidates all existing JWT tokens. All users will need to log in again.

## Service Management

```bash
# Status
sudo systemctl status openzfs-manager

# Logs (follow)
sudo journalctl -u openzfs-manager -f

# Restart
sudo systemctl restart openzfs-manager

# Stop
sudo systemctl stop openzfs-manager
```

## Nginx Configuration

The default config serves on port 80 without TLS. For production:

```bash
sudo vim /etc/nginx/sites-available/openzfs-manager
```

### Enabling HTTPS

1. Obtain TLS certificates (Let's Encrypt, self-signed, etc.)
2. Update the nginx config:

```nginx
server {
    listen 443 ssl http2;
    server_name zfs.example.com;

    ssl_certificate     /etc/letsencrypt/live/zfs.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/zfs.example.com/privkey.pem;

    # ... rest of config unchanged ...
}

server {
    listen 80;
    server_name zfs.example.com;
    return 301 https://$host$request_uri;
}
```

3. Test and reload: `sudo nginx -t && sudo systemctl reload nginx`

## Updating

```bash
# Pull new code
cd /path/to/source
git pull

# Re-run installer (idempotent)
sudo bash deploy/install.sh
```

The installer uses `rsync --delete` so removed files are cleaned up. The Python venv and npm install are incremental.

## Backup

### Database
```bash
# SQLite database
cp /opt/openzfs-manager/data/openzfs.db /backup/openzfs.db.$(date +%Y%m%d)
```

### Configuration
```bash
# Service config (contains JWT secret)
cp /etc/systemd/system/openzfs-manager.service /backup/

# Samba shares
cp /etc/samba/openzfs-shares.conf /backup/

# NFS exports
cp /etc/exports.d/openzfs.exports /backup/
```

## Troubleshooting

### Service won't start
```bash
journalctl -u openzfs-manager -n 50 --no-pager
```

### ZFS commands fail
```bash
# Check ZFS module is loaded
lsmod | grep zfs
sudo modprobe zfs

# Check command paths
which zpool zfs
```

### Database locked errors
SQLite can only handle one writer at a time. Under heavy load, consider increasing `command_timeout` or ensuring only one instance of the service runs.

### Samba shares not appearing
```bash
# Check include is in smb.conf
grep openzfs /etc/samba/smb.conf

# Test Samba config
testparm -s

# Check service status
systemctl status smbd
```

### NFS exports not working
```bash
# Check exports file
cat /etc/exports.d/openzfs.exports

# Re-export
sudo exportfs -ra

# Check active exports
sudo exportfs -v
```
